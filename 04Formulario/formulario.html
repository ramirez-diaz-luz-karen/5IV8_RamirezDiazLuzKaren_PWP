Todos los formularios en la web, manejan metodos para el envio de las diferente peticiones que realiza el usuario. Considerando la arquitectura cliente-servidor, el manejo de las peticiones se hacen por medio del protocolo http y actualmente por https, cada peticion debe ser atendida en tiempo de respuesta conforme al modelo TCP/IP. Los metodos principales de los formularios son

    • GET:

      Este metodo se encarga de enviar la informacion del formulario por medio de la salida estandar (conocido como url), del navegador, esto propicia que puedan existir inyecciones de codigo realizando violaciones a la seguridad del sistema.

      Post:

      Este metodo se encarga de enviar la información para poder registrar un cambio o un nuevo registro dentro del sistema enviandola igual por el método estandar pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo de la información


    • En resumen los métodos son::

    • Get: sirve para enviar y realizar operaciones de lectura o consulta
      Post: Sirve para enviar información de un recurso al servidor con el fin de registrarlos
      Delete: Sirve para eliminar un recusro especifico dentro del servidor, obteniendo una respuesta de actualización
      Put: Sirve para almacenar la información de un recurso particular y conectarse al servicio para realizar la actualización
      Head: Sirve para regresar el codigo de puesta de cualquier cabecera HTTP asociada a un recurso
